🐦 SeaGull MD Viewer — help

Using online — no install needed

You can bookmark and use the viewer directly from any browser without downloading anything:

https://aiagentview.github.io/seagull-md-viewer/seagull-md-viewer.html

Use the 📂 open folder button to open a folder of .md files as a browsable vault, or drag a single file onto the window to render it.

Privacy: Your files never leave your machine. The hosted URL only serves this HTML file — all file reading happens locally in your browser. If you see a browser prompt saying "Upload N files to this site?" when reopening a folder, that is a browser dialog (not this viewer) asking permission to restore access to a folder you previously opened. No data is uploaded or sent anywhere.

Top bar

Glyphs, left to right: ☆ favorite the open file (see below), ⛶ full-page reading (hides the sidebar and menus — only the rendered content remains; Esc or the floating ✕ top-right exits), 🔄 reload, ❓ this help. Two dropdowns set the color scheme and the reading font.

Favorites

  • With a file open, click ☆ in the top bar to star it; the glyph fills to a gold ★. Starred files appear in a Favorites section above Recent.
  • Favorites are the way to keep individual files handy — Recent mostly tracks the folders and links you open, not files clicked inside a tree.
  • Click a favorite to open it; the ★ on its row (or the top-bar star) un-stars it.
  • Starring snapshots the file's content, so a favorite reopens even when the folder isn't loaded and without the file:// flag. (It's a point-in-time copy; re-star to refresh.)

Color scheme & reading font

Two dropdowns in the top bar; both choices are remembered. The font (System / Serif / Book / Verdana / Mono — all built into the OS, nothing downloaded) applies only to the rendered content; the UI and code blocks are unaffected.

  • Midnight — the original dark navy/cyan scheme.
  • Paper — clean black-on-white, GitHub-style.
  • Dark — neutral charcoal grey, easy on the eyes.
  • Sepia — warm low-contrast tones for long reading.

Opening things

  • Drop a folder anywhere on this window — its .md, .html, and .pdf files appear as a tree in the sidebar. Drop a single file to render it directly.
  • 📂 open folder — same thing via a picker instead of dragging. Folders open this way load lazily — only the files you click are read (large folders open instantly). Recent then remembers just the files you've viewed; on a later reload, unopened files prompt you to re-pick the folder.
  • Paste a URL or path on the empty start page. End it with / to load it as a folder.
  • Parameters: seagull-md-viewer.html?file=/abs/path/foo.md renders one file; seagull-md-viewer.html?dir=/abs/path/folder loads a whole folder into the sidebar. Both also accept http(s):// URLs.

Sidebar

  • Tree — click a file to render it; folders collapse/expand. Hover a clipped name to see it in full.
  • Filter box — type to narrow the tree (matches anywhere in the path).
  • Resize — drag the divider between sidebar and content; double-click it to reset (the width is remembered). To hide the sidebar entirely, use ⛶ full-page mode in the top bar.

Recent

  • Folders, files and parameter loads land here automatically (newest first, capped at 15). ✕ removes one entry, clear empties the list.
  • Chrome/Edge: reopening a dropped folder is live — the browser asks once to re-allow access, then you see current content.
  • Brave/Safari: dropped folders reopen from a cached snapshot (marked “cached”). Re-drop the folder to refresh it.
  • Everything is stored locally in this browser profile (localStorage + IndexedDB). Clearing site data wipes it.

Folder location

Browsers hide the absolute path of dropped folders, so the viewer can’t know where files live on disk. Type the folder’s path once into the folder location box above the tree (e.g. /Users/you/recipes) — it’s saved with the recent entry. This unlocks:

  • Drag-out links for every file in the tree,
  • inline images in markdown (relative paths resolve to real files),
  • full-fidelity HTML (images and styles load).

Folders loaded via ?dir= need none of this — the path is already known.

Drag a file to another window

  • Drag a file row from the tree — or the filename in the top bar — and drop it on another browser window’s tab bar or address bar. That window opens the viewer on that file.
  • Dragging a folder name gives a ?dir= link that opens the whole folder.
  • Requires the folder location to be set (or a ?file=/?dir= load, where it’s automatic).

Obsidian-flavored markdown

Beyond standard Markdown + GFM (tables, task lists, strikethrough), these Obsidian extensions are supported — all built-in, no plugins:

  • Wikilinks [[note]], [[path/note]], [[note#heading|alias]] — click to open the target note (and scroll to the heading). Resolves by path or basename within the loaded folder; unresolved links say "note not found".
  • Embeds ![[image.png]] — shown when the folder location is known (resolved against it).
  • Callouts > [!note], [!tip], [!warning], … — styled boxes; add -/+ ([!tip]-) for a collapsible one.
  • Frontmatter (YAML --- block) → a compact properties table.
  • Highlight ==text==, comments %%hidden%%, footnotes [^1], and custom task states - [/] - [-] - [>].

Wikilinks resolve only when a folder/vault is loaded (like Obsidian needing the vault open). Math $…$ and Mermaid are intentionally not supported — they'd require heavyweight libraries and break the single-file, offline design.

HTML & PDF files

HTML files show up in the tree (marked ‹html›) and render in an isolated frame with scripts disabled. With the folder location set, the frame loads the real file so its images and CSS work; otherwise it renders from the cached copy without them.

PDF files (marked ‹pdf›) render in your browser's built-in PDF viewer — no extra library is bundled. They work from a dropped file, the folder location, the server, or ?file=; a PDF in a dropped folder is held in memory so it previews even on Brave/Safari.

Running over a local server (optional)

Drag-and-drop needs nothing. But to use ?file=/?dir= without a browser flag — and to make relative images, ?dir= folder browsing, and drag-out links all work — serve the folder over a tiny local web server. Any static server works; the simplest, with no install, is Python's:
python3 -m http.server 8001 --bind 127.0.0.1
then open http://localhost:8001/seagull-md-viewer.html. The folder you serve becomes the web root, so keep the viewer and your content under it. (npx serve, php -S localhost:8001, … work too.) Bonus: this mirrors how the viewer behaves once deployed, so it doubles as a pre-publish test.

Caveats

  • Loading markdown via ?file=/?dir= with file:// paths needs the browser launched as
    open -na "Brave Browser" --args --allow-file-access-from-files
    or a local server (python3 -m http.server). Drag-and-drop never needs this.
  • Brave disables the File System Access API by default, so folder reopening falls back to cached snapshots there.

Customizing — add your own theme

This is one self-contained HTML file; everything is editable inside it. A theme is just a set of CSS-variable values — no style rules change. To add one:

  1. Near the top of the <style> block, find the comment
    /* ===== Color schemes — :root is "midnight"… */
  2. Copy any existing html[data-theme="…"] block (e.g. the paper one) and rename it, say html[data-theme="forest"].
  3. Change its values — --app-bg, --panel, --text, --link, --code-bg, --code-text, and the rest. Keep all the variables; a missing one falls back to Midnight and looks broken.
  4. Add a choice to the dropdown: in <select id="themesel"> add <option value="forest">Forest</option>.
  5. Register the key in two JS lists: the THEMES array (const THEMES = [… , 'forest']) and the matching array in the small anti-flash script in <head> (so it applies before first paint).

Customizing — add your own font

  1. Find the comment /* ===== Reading fonts … */ and add a line like
    html[data-font="charter"] { --read-font: Charter, Georgia, serif; }
  2. Use only fonts already installed on the system, written as a fallback stack — that keeps the file self-contained (no downloads).
  3. Add a choice to <select id="fontsel">:
    <option value="charter" style="font-family:Charter,serif">Charter</option>
  4. Register the key in the FONTS array and the anti-flash font list in <head>.

The reading font applies only to #content (the rendered markdown); the UI chrome and code blocks are untouched. Theme and font are independent — any combination works.
